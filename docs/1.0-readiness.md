## Known Limitations

1. **XLSX file upload parsing** — Foundation exists (`xlsx` dependency, `summarizeForAgent` structure), but the file upload handler in the UI needs to pass XLSX bytes to the parser. Currently only CSV/TSV/JSON text input generates source keys.

2. **E2E test data-testid attributes** — The killer-flow E2E test references `[data-testid=...]` selectors that may not exist on components yet. These need to be added to: welcome modal, editor textarea, template picker, convert button, preview iframe, verification receipt, export menu.

3. **Mock agent fixture realism** — The data-brief fixture has `pf-src` annotations, but the exact key format (`rows[].department`) may need tuning to match what the verification engine expects after real agent output.

4. **Verification score for mock output** — The mock fixture's source-key coverage may not be 100% because KPI card values (avg, max, min) don't have direct `rows[]` keys. The verification will show this accurately — which is actually good for demonstrating the feature.

5. **No runtime testing done** — Because the environment lacks `pnpm install`, no tests have been executed. The user must run the test suite locally.
